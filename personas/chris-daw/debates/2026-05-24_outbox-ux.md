# Debate: Outbox UX — Productivity Focus

**Date:** 2026-05-24
**Participants:** Chris Daw (RCIC, R409583) + Claude
**Focus:** Cutting admin paperwork time in half

---

## The Problem

Chris spends 3 hours/week on templated documents (cover letters, retainer agreements). Each takes 45 minutes of copy-paste-edit-export. 90% identical every time.

## Productivity Math

| Task | Current | Target | Savings |
|---|---|---|---|
| Single cover letter | 45 min | 5 min | 40 min |
| 4 letters/week | 3 hrs | 20 min | 2.5 hrs/week |
| Annual batch (40 clients) | 10 hrs | 10 min | ~10 hrs |

## The 5 Templates (80% of output)

1. **IRCC Cover Letter** — accompanies every application submission
2. **Retainer Agreement** — new client engagement contract
3. **Document Request Letter** — asking clients for missing items
4. **Employer Support Letter** — LMIA template for employers
5. **Client Status Update** — periodic progress report

## Chris's Decision: 3-Step Pipeline, No Editor

### Step 1: Template Picker
Pick from 5 templates. Each shows a preview and lists the auto-fill fields.

### Step 2: Auto-Fill Review
Select a client → all fields populate from their profile (name, DOB, passport, application number, dates). Edit the free-text paragraph for the specific situation. No formatting toolbar, no fonts, no rich text.

### Step 3: Generate
One button → professional PDF. Download or send directly. Document saved to sent history linked to client.

## Key Decisions

- **NOT a word processor** — it's a template-to-PDF pipeline
- **Auto-fill is the productivity gain** — not the editor
- **5 templates cover 80%** — the other 20% stays in Word
- **Batch generation** — select N clients × 1 template = N PDFs
- **Sent history = audit trail** — timestamped, client-linked, searchable

## Verdict

Build the template pipeline. No TipTap, no Slate, no rich text editor. A form with smart defaults that generates PDFs. The 80/20 rule: handle the 80% that's templated, let Word handle the 20% that's custom.
